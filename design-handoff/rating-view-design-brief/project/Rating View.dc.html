Continuum / Design system
Reference sheet · v0.1
Continuum — rating instrument tokens
A quiet, research-grade system. Neutral greys carry all chrome; saturated colour is reserved for the rating scale, so the participant reads colour as meaning. Every surface is a CSS custom property with a dark and light value.
01Colour tokens
Neutral UI. Dark is the default (better for watching video); light is required and derived.
TokenDarkLightUsage
--bg #101216 #eceef1 App background
--surface #171a20 #ffffff Panels, header, cards
--surface-2 #1e222a #e8eaee Insets, tracks, controls
--raised #20252e #ffffff Popovers, value readout
--line #2a2f39 #dbdee3 Borders, ticks, grid lines
--fg #e6e8ec #191c22 Primary text
--fg-2 #9aa1ad #525963 Secondary text, labels
--fg-3 #6b7280 #8a919b Tertiary, ticks, hints
--accent #5b8def #5b8def Handle / point, selection
Chrome stays neutral so the scale can be loud. Accent is the only chromatic UI colour, and only on the live control — everything else is grey.
02Rating scale palettes
Defaults are perceptually-uniform and colourblind-safe. The classic red–yellow–green scale is excluded — it fails the most common CVD.
